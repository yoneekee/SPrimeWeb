# 반도체 정밀 기기 제조 및 통합 재고 관리 시스템 (S-PRIME ERP)

**S-PRIME ERP : S**emiconductor **P**recision **R**esource & **I**nventory **M**anagement **E**RP

---

# 1. 프로젝트 개요

본 프로젝트는 반도체 정밀 부품 및 장비 제조업의 특성을 반영하여, **원재료 조달부터 BOM(Bill of Materials) 기반의 생산, 그리고 최종 제품 출고에 이르는 전 과정을 통합 관리**하는 Enterprise Resource Planning(ERP) 시스템입니다. 전표 기반의 승인 프로세스를 통한 업무 통제, 실시간 수불(Inventory Flow) 추적, 그리고 비즈니스 트랜잭션과 회계 분개의 자동 연동을 통한 데이터 정합성 확보를 목표로 합니다.

# 4. 주요 기능 요건 (Functional Requirements)

### 4.1 구매 및 제조 실행 (Procurement & Manufacturing)

- **BOM 기반 자동 전개:** 완제품 생산 계획 수립 시 하위 부품 리스트를 자동 추출하여 과부족 현황을 시각화.
- **분납 관리(Partial Delivery):** 분납 시에는 `stock_spec_history`에 입고 이력만 누적하여 관리 효율성 증대. 최종 입고 종료(`P04`) 시점에 누적 수량을 합산하여 매입채무 확정 분개를 실행.

### 4.2 물류 및 출고 관리 (Logistics)

- **적송 재고(In-Transit) 관리:** 창고를 떠났으나 매출로 인식되기 전 단계인 '적송품' 계정을 별도 관리하여 자산 정합성 유지.
- **FIFO 기반 LOT 추적:** 반도체 부품의 특성을 고려하여 선입선출 기반의 LOT 지정 출고 기능 지원.

### 4.3 회계 연동 및 리포팅 (Accounting & Reporting)

- **자동 분개 엔진:** 전표 상태 변경(Trigger)에 따라 사전 정의된 계정과목으로 분개 데이터를 자동 생성.
- **재무제표 Export:** 대차대조표(B/S), 손익계산서(P/L)를 일본 표준 양식 및 PDF로 출력.

# 5. 기능 요건 정의

## 5.1 단계별 기능 및 데이터 흐름 요건

### [1] 신청 및 승인

1. **신청:** 로그인 유저가 전표를 작성하고 결재를 요청 (`신청중`).
2. **승인:** 지정된 승인 자격자들이 모두 승인해야 다음 단계로 진입.
   - **부인:** 승인자 중 한 명이라도 거절 시 전표 종료 (`부인중`).
   - **반려:** 승인 완료 후 **발주 전**에 제조 담당자가 신청자에게 수정 요청 (`반려중`).

### [2] 제조/구매 실행

1. **견적(見積):** 제조 담당자가 견적 정보를 입력.
   - **재고:** 품목 마스터의 **예상 재고** 증가 (입고 예정 수량 확보).
2. **발주(発注):** 업체 확정 및 발주 전표 생성.
   - **회계:** **[차변] 미착품(자산) / [대변] 미지급금(부채)**
3. **분납(分納):** 여러 번에 나누어 입고 정보를 등록. 워크플로우 이력에 기록 누적.
   - **회계:** 분납 시마다 발생 금액만큼 **회계 분개 테이블**에 임시 기록.
4. **입고(入庫):** (분납 누계 금액/수량 == 발주 금액/수량) 일치 시 버튼 활성화.
   - **회계:** **[차변] 원재료(자산) / [대변] 매입채무** 확정 등록.

### [3] 검수 및 수불 완료

1. **검수(検収):** 검수 담당자가 검수 결과를 기록.
   - **재고:** 품목 마스터의 **예상 재고 차감**, **실재고 증감** 반영.
   - **수불:** 수불 이력 테이블에 최종 '입고 완료' 트랜잭션 기록.

### [4] 재무제표 반영 및 서류 출력

- 검수 종료 후 재무제표에 자동 반영되며, 각종 서류(청구서, 발주서)의 출력이 가능.

# 6. 데이터베이스 설계 핵심 (DB Architecture)

- **트랜잭션 무결성:** 검수 종료(`I00`) 및 매출 확정(`T03`) 시, 다중 테이블(수량, 이력, 분개) 업데이트를 단일 트랜잭션으로 묶어 원자성(Atomicity)을 보장.
- **Concurrency Control:** 재고 차감 시 **비관적 잠금(Pessimistic Lock)** 및 `item_id` 정렬 순서에 따른 잠금 획득으로 데드락(Deadlock) 방지 및 데이터 정확도 확보.

# 8. 화면 정의

### 8.1 생산 출고 전표 일람 화면 ★신규

- 제조 구매(PRODUCTION) 및 출고(SHIPMENT) 전표를 **통합적으로 검색·조회**하는 화면
- 전표 구분, 상태, 기간, 담당자 등 다양한 조건으로 필터링하여 전표 현황을 한눈에 파악

#### 검색 조건 영역

| **화면 요소** | **설명** | **요소 종류** | **비고** |
|---------------|----------|---------------|----------|
| **전표 구분** | 제조 구매(PROD) / 출고(SHIP) / 전체 선택 | 콤보박스 | 기본값: 전체 |
| **전표 상태** | S00~I00, T01~T04 중 선택 | 콤보박스 | 상태 코드와 명칭 병기 |
| **의뢰일(FROM)** | 조회 시작일 | 날짜 선택 | - |
| **의뢰일(TO)** | 조회 종료일 | 날짜 선택 | - |
| **의뢰자** | 전표 기안자별 필터링 | 콤보박스 | 직원 마스터 연동 |
| **승인자** | 결재 승인자별 필터링 | 콤보박스 | 직원 마스터 연동 |
| **구매/제조 담당** | 실무 담당자별 필터링 | 콤보박스 | 직원 마스터 연동 |
| **전표번호/거래처** | 키워드 검색 (전표 번호 또는 거래처명) | 텍스트 입력 | 부분 일치 검색 |
| **초기화** | 모든 검색 조건을 기본값으로 초기화 | 버튼 | - |

#### 전표 일람 영역

| **컬럼명** | **설명** | **데이터 소스** | **비고** |
|------------|----------|-----------------|----------|
| **전표 번호** | 전표의 고유 식별 번호 | `tran_history.slip_no` | 클릭 시 상세 화면으로 이동 |
| **전표 구분** | 제조 구매 / 출고 구분 표시 | `tran_history.slip_type` | 뱃지 형태로 표기 |
| **전표일자** | 전표 작성일 | `tran_history.slip_date` | YYYY/MM/DD |
| **의뢰자** | 전표 기안자 성명 | `emp_mst` JOIN | - |
| **부서** | 기안자 소속 부서 | `emp_mst.dept_code` | 범용 마스터 연동 |
| **승인자** | 결재 승인자 성명 | `workflow` JOIN | - |
| **담당자** | 구매/제조 실무 담당자 | `workflow` JOIN | - |
| **현재 상태** | 전표의 현재 진행 상태 | `tran_history.status_code` | 색상 뱃지로 시각화 |
| **거래처** | 관련 거래처 명칭 | `customer_mst` JOIN | - |
| **합계 금액** | 전표 총 합계 (JPY) | `tran_history.total_amount` | ¥ 통화 서식 |
| **품목 수** | 전표에 포함된 품목 건수 | `tran_detail_history` COUNT | - |
| **상세** | 해당 전표의 상세 화면으로 이동 | 버튼/아이콘 | PROD → 8.2, SHIP → 8.3 |

#### PDF 문서 구성 항목

**A. 발주서 (Purchase Order / 発注書)**
- **상단:** 자사 정보(회사명, 주소, 연락처), 발주번호, 발행일, 공급사 귀하
- **본문:** 품목명, 규격, 수량, 단가, 공급가액, 소비세(10%), 합계금액
- **하단:** 납기일, 납품 장소, 결제 조건, 자사 직인란(Hanko)

**B. 청구서 (Invoice / 請求書)**
- **상단:** 자사 정보 및 은행 계좌 정보(입금처), 고객사 정보, 청구번호
- **본문:** 출고 내역별 품목, 수량, 금액, 소비세 별도 표기
- **특이사항:** "하기 금액을 청구합니다"라는 표준 문구 및 지불 기한 명시

#### 문서 출력 핵심 비즈니스 로직

1. **세율 계산 (Tax Logic):**
   - 통상 10%의 소비세를 적용
   - 각 행별로 계산할지, 전체 합계에서 계산할지에 따라 단수 처리(절사, 반올림)가 달라지므로 **범용 마스터**의 설정값을 따르도록 설계

| **항목** | **설명** | **요소 종류** | **필수** | **비고** |
|----------|----------|---------------|----------|----------|
| **품목 코드** | 고유 식별 코드 (중복 불가) | 텍스트박스 | **Y** | 등록 후 수정 불가 권장 |
| **품목 명칭** | 국문/일문/영문 명칭 (현지화 고려) | 텍스트박스 | **Y** | - |
| **품목 분류** | RAW, SEMI, FIN 중 선택 | 콤보박스 | **Y** | 수불 로직의 트리거 |
| **상세 규격** | 치수, 재질, 전압 등 사양 정보 | 멀티라인 | N | - |
| **관리 단위** | EA, SET, KG, BOX 등 | 콤보박스 | **Y** | 범용 마스터 `UNIT` 참조 |
| **표준 단가** | 회계상 평가 및 원가 산출 기준가 | 숫자 입력 | **Y** | `decimal` 타입 처리 |
| **기본 계정과목** | 해당 품목 입출고 시 자동 분개될 계정 | 콤보박스 | **Y** | `1101: 원재료` 등 연결 |

| **사용 여부** | 단종되거나 더 이상 쓰지 않는 품목 처리 | 토글/체크 | **Y** | 기본값: 사용(Y) |
