# 半導体精密機器製造および統合在庫管理システム（S-PRIME ERP）

**S-PRIME ERP : S**emiconductor **P**recision **R**esource & **I**nventory **M**anagement **E**RP

---

# 1. プロジェクト概要

本プロジェクトは、半導体精密部品および装置製造業の特性を反映し、**原材料の調達からBOM（Bill of Materials）に基づく生産、そして最終製品の出庫に至る全工程を統合管理する**Enterprise Resource Planning（ERP）システムである。伝票ベースの承認プロセスによる業務統制、リアルタイムの受払（Inventory Flow）追跡、およびビジネストランザクションと会計仕訳の自動連動によるデータ整合性の確保を目的とする。

|----------|------------|--------------------------|--------------------------|------------------------------------------|----------------|
| **Common** | **S00** | Draft | 作成中 | 伝票起案の作成段階（初期状態） | 申請者 |
| | **S01** | Pending | 申請中 | 決裁要求処理完了（申請中） | 申請者 |
| | **A00** | In-Review | 承認中 | 決裁権者による審査進行中 | 承認者 |
| | **A01** | Approved | 承認済 | 最終承認完了（全承認者の承認完了時に自動更新） | 承認者 |
| | **A02** | Rejected | 否認中 | 決裁却下（伝票無効化、再申請不可） | 承認者 |
| **Production** | **P00** | Returning | 差戻中 | 製造・購買担当者が申請者へ修正依頼（修正後に再申請・再承認が必要）/ 発注以降は不可 | 製造・購買担当者 |
| | **P01** | Quoted | 見積中 | 見積確定および**予定在庫（Plan Qty）確保** | 製造・購買担当者 |
| | **P02** | Ordered | 発注済 | 発注確定。**[借] 未着品 / [貸] 未払金** 発生 | 製造・購買担当者 |
| | **P03** | Partial | 分納中 | 分納入庫進行。**数量のみ記録（会計処理未発生）** | 製造・購買担当者 |
| | **P04** | Received | 入庫済 | 最終入庫完了（**買掛金**確定）。**[借] 原材料 / [貸] 未着品**、**[借] 未払金 / [貸] 買掛金** 一括振替 | 製造・購買担当者 |
| **Inspection** | **I00** | Accepted | 検収済 | 最終検収合格。**実在庫（Stock）反映および予定在庫差引**、実在庫増減および**棚卸減耗損**処理 | 検収担当者 |
| **Logistics** | **T01** | Transit | 積送中 | 製品出荷および配送開始。**[借] 積送品 / [貸] 完成品** | 製造・購買担当者 |
| | **T02** | Delivered | 出庫済 | 顧客引渡完了。実在庫差引 | 製造・購買担当者 |
| | **T03** | Invoiced | 売上済 | 売上確定。**[借] 売掛金 / [貸] 売上高**、**[借] 売上原価 / [貸] 積送品** | 製造・購買担当者 |
| | **T04** | Adjusted | 在庫調整済 | 在庫減耗・増額調整（例外処理） | 製造・購買担当者 |

# 4. 主要機能要件（Functional Requirements）

### 4.1 購買および製造実行（Procurement & Manufacturing）

- **BOM基盤自動展開:** 完成品の生産計画策定時に下位部品リストを自動抽出し、過不足状況を可視化。
- **分納管理（Partial Delivery）:** 分納時には `stock_spec_history` に入庫履歴のみ蓄積し管理効率を向上。最終入庫完了（`P04`）時点で累積数量を合算し、買掛金確定仕訳を実行。

### 4.2 物流および出庫管理（Logistics）

- **積送在庫（In-Transit）管理:** 倉庫を出たが売上として認識される前の段階にある「積送品」勘定を個別管理し、資産整合性を維持。
- **FIFO基盤LOT追跡:** 半導体部品の特性を考慮した先入先出基盤のLOT指定出庫機能を提供。

### 4.3 会計連動およびレポーティング（Accounting & Reporting）

- **自動仕訳エンジン:** 伝票ステータス変更（Trigger）に基づき、事前定義された勘定科目への仕訳データを自動生成。
- **財務諸表Export:** 貸借対照表（B/S）、損益計算書（P/L）を日本標準様式およびPDFで出力。

# 5. 機能要件定義

## 5.1 段階別機能およびデータフロー要件

### [1] 申請および承認

1. **申請:** ログインユーザーが伝票を作成し決裁を要求（`申請中`）。
2. **承認:** 指定された承認資格者が全員承認すると次のステップへ進む。
   - **否認:** 承認者のうち一人でも拒否した場合、伝票を終了（`否認中`）。
   - **差戻:** 承認完了後、**発注前**に製造担当者が申請者へ修正を依頼（`差戻中`）。

### [2] 製造・購買実行

1. **見積（見積）:** 製造担当者が見積情報を入力。
   - **在庫:** 品目マスターの**予定在庫**を増加（入庫予定数量の確保）。
2. **発注（発注）:** 業者確定および発注伝票の作成。
   - **会計:** **[借方] 未着品（資産） / [貸方] 未払金（負債）**
3. **分納（分納）:** 複数回に分けて入庫情報を登録。ワークフロー履歴に記録を蓄積。
   - **会計:** 分納ごとに発生金額分を**会計仕訳テーブル**に仮記録。
4. **入庫（入庫）:**（分納累計金額・数量 == 発注金額・数量）一致時にボタンを活性化。
   - **会計:** **[借方] 原材料（資産） / [貸方] 買掛金** 確定登録。

### [3] 検収および受払完了

1. **検収（検収）:** 検収担当者が検収結果を記録。
   - **在庫:** 品目マスターの**予定在庫を差引**、**実在庫の増減**を反映。
   - **受払:** 受払履歴テーブルに最終「入庫完了」トランザクションを記録。

### [4] 財務諸表反映および書類出力

- 検収終了後に財務諸表へ自動反映され、各種書類（請求書、発注書）の出力が可能。

## 5.2 会計勘定科目

| **区分** | **勘定科目名** | **コード** | **説明** |
|----------|----------------|------------|----------|
| **資産** | 原材料 | 1101 | 外部から購入した基礎部品および原材料 |
| | 半製品 | 1102 | 組立中のモジュールや加工された中間部品 |

# 6. データベース設計の要点（DB Architecture）

- **トランザクション無結性:** 検収終了（`I00`）および売上確定（`T03`）時、複数テーブル（数量、履歴、仕訳）の更新を単一トランザクションにまとめ、原子性（Atomicity）を保証。
- **Concurrency Control:** 在庫差引時に**悲観的ロック（Pessimistic Lock）**および `item_id` のソート順に従ったロック取得により、デッドロック（Deadlock）を防止しデータの正確性を確保。

| **品目概要** | 「○○ 外N件」形式で表記 | `tran_detail_history` | - |
| **合計金額** | 税込の最終合計金額 | `total_amount` | - |
| **発行状況** | 出力済みかどうかの表示 | システムロジック | 重複出力防止用 |
| **出力** | 該当件のPDFプレビューおよびダウンロード | アイコン/ボタン | 単件出力 |

#### PDF文書構成項目

**A. 発注書（Purchase Order / 発注書）**
- **上段:** 自社情報（会社名、住所、連絡先）、発注番号、発行日、供給者宛
- **本文:** 品目名、規格、数量、単価、供給価額、消費税（10%）、合計金額
- **下段:** 納期、納品場所、決済条件、自社職印欄（Hanko）

**B. 請求書（Invoice / 請求書）**
- **上段:** 自社情報および銀行口座情報（振込先）、顧客情報、請求番号
- **本文:** 出庫明細別の品目、数量、金額、消費税別途表記
- **特記事項:** 「下記金額をご請求申し上げます」の標準文言および支払期限の明示

#### 文書出力コアビジネスロジック

1. **税率計算（Tax Logic）:**
   - 通常10%の消費税を適用
   - 各行別に計算するか、全体合計から計算するかにより端数処理（切捨て、四捨五入）が異なるため、**汎用マスター**の設定値に従う設計
